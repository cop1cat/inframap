# InfraMap Viewer — ТЗ

Статический сайт для интерактивного просмотра `infra.json`, сгенерированного CLI-утилитой `inframap`.

## Цели

- Открыть `infra.json` в браузере и сразу понять структуру системы.
- Никакого бэкенда, аутентификации, редактирования. Только чтение и навигация.
- Хостится на `cop1cat.github.io/inframap` через GitHub Pages.

## Не-цели

- Редактор графа в UI.
- Хранение/синхронизация на сервере.
- Авторизация, мульти-пользовательность.
- Экспорт в форматы кроме SVG.

---

## Стек

- **Cytoscape.js** + layout **cose-bilkent** — единственная зрелая связка с поддержкой compound nodes (вложенные группы) и force-directed раскладки одновременно.
- **[cytoscape-svg](https://github.com/kaluginserg/cytoscape-svg)** — экспорт в SVG (поддерживает compound nodes; fallback на `cy.png({ scale: 2 })`).
- **TypeScript** + **Vite** — сборка статики.
- **Svelte** — UI-фреймворк (быстрый старт, маленький бандл, нативная реактивность для панелей и фильтров).
- **[tippy.js](https://atomiks.github.io/tippyjs/)** — tooltips (готовое, стабильное, поддерживает HTML-контент).
- **[Lucide](https://lucide.dev/)** — иконки (owner, типы вызовов в легенде). Чипсы для тегов — свой минимальный CSS.
- Без серверной части. Деплой — `npm run build` → `dist/` → push в ветку `gh-pages`.

Принцип: всё графовое (раскладка, compound nodes, рендеринг рёбер, zoom/pan) — на готовых библиотеках. Свои велосипеды только там, где их нет.

## Загрузка `infra.json`

Три способа, все клиентские, без бэкенда:

1. **Drag-n-drop** файла в окно. По всей странице активна drop-зона; при перетаскивании показывается оверлей.
2. **Paste** (`Ctrl+V` / `Cmd+V` на странице) — вставка JSON из буфера обмена.
3. **`#data=<gzip+base64url>`** — JSON упакован прямо в URL fragment. Используется для шаринга ссылок без хостинга. Распаковка через нативный `DecompressionStream` API (без внешних либ типа `pako`). Fragment не уходит на сервер GitHub Pages.

**LocalStorage:** последний загруженный JSON сохраняется и автоматически восстанавливается при следующем открытии (если нет `#data=`). Кнопка «Clear» в настройках чистит storage.

**Шаринг:** кнопка **«Share»** в UI кодирует текущий граф через `CompressionStream('gzip')` → `base64url` → копирует в буфер ссылку вида `https://cop1cat.github.io/inframap#data=...`. Для типичной инфры (50 сервисов ≈ 5-8 KB JSON) ссылка получается ~1.5-2 KB; до 500 сервисов укладывается в ~10-15 KB. Если граф слишком большой — UI показывает лимит и предлагает поделиться файлом напрямую.

**Валидация на входе:** `schema_version` обязателен. Если `schema_version != 1` — показать понятную ошибку «этот вьюер поддерживает только schema v1, ваш файл v{N}». JSON Schema (`schema/v1.json`) опционально подгружать для строгой валидации в dev-режиме.

## Визуальная модель

Идея — **«граф знаний как в Obsidian»**: спокойный фон, ничего не кричит, контекст появляется по взаимодействию.

### Дефолтное состояние (ничего не выделено)

- Force-directed раскладка через `cose-bilkent`.
- Группы — compound nodes с подписью `label` и тонкой рамкой; цвет рамки = `group.color` или дефолтный (тема-зависимый серый).
- Сервисы — круглые ноды с подписью `label` под нодой.
- Все рёбра — **полупрозрачные** (~0.2 opacity), нейтрального цвета. Никакой анимации.
- Нет различия по типу вызова визуально, кроме линии (см. ниже).

### Стили рёбер по `type`

| `type`    | Стиль линии        |
|-----------|--------------------|
| `sync`    | Сплошная           |
| `async`   | Пунктир            |
| `event`   | Точечная           |
| `unknown` | Серая сплошная     |
| `broken`  | **Красная** пунктирная (флаг `broken: true` в JSON) |

Стрелка на конце — у всех, направление = caller → callee.

### Подсветка по выделению

При **клике на сервис**:
- Сам сервис — opacity 1, акцентный обвод.
- Все инцидентные ему рёбра (входящие и исходящие) — opacity 1.
- Сервисы на других концах этих рёбер — opacity 1.
- Остальной граф приглушается (~0.15 opacity).
- На подсвеченных рёбрах включается **анимация направления**: вдоль ребра от caller к callee движутся точки/штрихи (через CSS-анимацию `stroke-dashoffset` или Cytoscape edge animations). В дефолтном состоянии анимации нет — это спойлер контекста только для выделенного.

При **клике на ребро**:
- Само ребро + два endpoint-сервиса подсвечиваются по тем же правилам.
- Анимация направления включается на этом ребре.

При **клике на группу** (compound node):
- Группа подсвечивается, все её прямые дети-сервисы подсвечиваются, остальное приглушается.

При **клике на пустое место** — сброс выделения, возврат в дефолт.

**Hover** (наведение, без клика):
- Лёгкая подсветка ноды/ребра под курсором (opacity 0.6, не 1) — подсказка «здесь кликабельно».
- Tooltip показывается через ~300ms hover, см. ниже.

### Tooltip на сервисе

Появляется на hover (с задержкой) и при выделении. Содержит:
- `label` (большим), `id` (моноширинным, мелким).
- `description` если есть.
- `owner` если есть.
- Чипсы по `tags` (key=value).
- Список `links` — кликабельные ссылки в новой вкладке.

### Клик на сервис — действие

- **Один клик** — выделение (см. выше).
- **Двойной клик** — открыть `links.docs` в новой вкладке, если есть. Если нет — ничего.

## Stats-панель

Боковая панель (toggle справа) со статистикой по загруженной инфре. Не аналитика сайта — статистика по самой карте.

- **Сводка**: всего сервисов, всего групп, всего связей.
- **По типам вызовов**: счётчик `sync` / `async` / `event` / `unknown`.
- **Broken links** — список (caller → ghost), клик подсвечивает связь на графе.
- **Без owner** — список сервисов, клик подсвечивает на графе.
- **Без description** — то же самое.
- **Топ owner** — таблица `owner: count`, клик фильтрует граф по этому owner.
- **Распределение по группам** — `group: services_count`, клик подсвечивает группу.

Клик по любой строке = фильтр/подсветка графа (как клик по самим элементам).

## Поиск и фильтры

**Панель сверху** (свернутая по умолчанию, разворот по `Ctrl+K` или клику):

- **Поиск**: одна строка, ищет по `label`, `id`, ключам и значениям `tags`, `owner`. Совпадения подсвечиваются (как выделение); остальное приглушается. Очистка — Esc.
- **Фильтр по тегам**: чипсы со всеми уникальными парами `tag:value`, multi-select. Скрывает несоответствующие сервисы (приглушает до невидимости, рёбра к ним тоже).
- **Фильтр по owner**: dropdown с уникальными значениями.

Фильтры комбинируются через AND. Сброс — кнопка «Reset».

## Темы

Тогглы **dark/light**, состояние сохраняется в localStorage. Дефолт — system preference (`prefers-color-scheme`).

Цветовая палитра отдельно для светлой и тёмной темы. `group.color` из JSON — это «акцент» группы, отображается на рамке независимо от темы.

## Экспорт

Кнопка **«Export as SVG»**. Использует `cytoscape-svg`. Fallback — `cy.png({ scale: 2 })` для PNG. SVG — для вставки в README. Состояние выделения и фильтров не учитывается; экспортируется текущая видимая раскладка в дефолтном стиле.

Кнопка **«Share»** — см. раздел «Загрузка». Копирует ссылку с встроенным `#data=...` в буфер.

## Пустой экран (ничего не загружено)

Центрированный блок с тремя способами загрузки:

```
┌───────────────────────────────────────┐
│       📁  Drop infra.json here        │
│                                       │
│       or paste with Ctrl+V            │
│                                       │
│       [ Try with sample ]             │
└───────────────────────────────────────┘
```

Кнопка **«Try with sample»** грузит встроенный мини-пример (3-5 сервисов в 1-2 группах), чтобы первый посетитель сразу увидел, как работает граф.

Ссылка на CLI и формат YAML — внизу страницы.

## Производительность

- Целевой размер: до **500 сервисов**, до **2000 рёбер** — должно крутиться плавно (60 fps на современном железе).
- На больших графах: автоматически отключать анимации направления, сохраняя только статичные стили.
- Layout пересчитывается один раз при загрузке. После — `cy.layout` не запускается, чтобы пользователь мог перетаскивать ноды без скачков.

## Что не делаем

- Никакого бэкенда, БД, аутентификации.
- Никакой сторонней аналитики (Plausible, GA и пр.).
- Никакого polling/watch — `infra.json` обновляется ручным reload или новой загрузкой.
- Никакого редактирования графа в UI — только просмотр.
